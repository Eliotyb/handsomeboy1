# 🔬 Deep Research - 深度调研专家

## 技能概述

提供系统化的深度调研能力，支持多源信息采集、自动去重、交叉验证、结构化输出专业级研究报告。

## 核心能力

### 1. 多源信息采集
- **搜索引擎**: Google/Bing/Baidu/DuckDuckGo 多引擎并行
- **学术资源**: 知网/Google Scholar/arXiv 论文检索
- **行业平台**: 艾瑞/易观/36氪/虎嗅 行业报告
- **社交媒体**: 微博/知乎/Reddit/Twitter 舆情
- **官方渠道**: 官网/白皮书/GitHub/文档站
- **新闻资讯**: 各大门户新闻聚合

### 2. 智能信息处理
- 自动去重与合并相似内容
- 信息可信度评估与标注
- 时间过滤（优先最新信息）
- 多语言翻译与统一
- 结构化数据提取

### 3. 专业报告生成
- Markdown 格式输出
- 包含目录/摘要/正文/结论/参考文献
- 支持图表数据可视化建议
- 可导出 PDF/Word/PPT

## 调研类型模板

### 📊 行业分析报告
```
输出结构:
1. 行业概述（定义/规模/发展阶段）
2. 市场格局（头部玩家/市场份额）
3. 发展趋势（技术/政策/消费）
4. 痛点与机会
5. 投融资情况
6. 未来预测
7. 参考来源
```

### 🎯 竞品分析报告
```
输出结构:
1. 竞品筛选标准
2. 基本信息对比表
3. 产品功能对比矩阵
4. 商业模式分析
5. 优劣势SWOT分析
6. 用户评价舆情
7. 差异化机会点
```

### 🔧 技术选型报告
```
输出结构:
1. 需求背景与约束
2. 候选方案列表
3. 多维度评分对比（性能/生态/成本/学习曲线）
4. POC验证建议
5. 风险评估
6. 最终推荐 + 理由
7. 迁移路径
```

### 📚 专题资料汇编
```
输出结构:
1. 调研问题定义
2. 信息来源清单
3. 核心发现要点
4. 详细资料摘录（附出处）
5. 开放问题与待确认项
6. 下一步行动建议
```

```
└─────────────────────────────────────────────┘
```

## 使用示例

### 示例1：行业调研
```
用户: 调研一下2025年中国的AI大模型应用市场
执行:
1. Phase1: 确认范围 - B端应用/C端应用? 垂直领域?
2. Phase2: 搜索策略 - "2025 AI大模型市场规模"/"大模型落地案例"/"AI应用投融资"
3. Phase3: 采集 - 艾瑞报告/IDC数据/36氪报道/厂商官网
4. Phase4: 分析 - 交叉验证数据一致性, 标注可信度
5. Phase5: 输出 - 完整行业分析报告(3000-5000字)
```

### 示例2：技术选型
```
用户: 我们项目要选前端框架，Vue还是React?
执行:
1. 收集: GitHub stars/npm下载量/社区活跃度/招聘需求
2. 对比: 学习曲线/生态系统/性能基准/团队匹配度
3. 验证: 查看最新版本发布/已知issue/ roadmap
4. 输出: 选型评分表 + 最终推荐理由
```

## 质量保证

| 维度 | 标准 |
|------|------|
| **信息新鲜度** | 优先近6个月信息，超1年标注"历史参考" |
| **来源多样性** | 至少3个独立来源交叉验证关键结论 |
| **可追溯性** | 每个论点标注原始出处URL |
| **客观中立** | 同时呈现正反观点，不预设立场 |
| **完整性检查** | 覆盖用户提出的所有关注点 |

## 注意事项

- ⚠️ 敏感话题需遵守内容安全规范
- ⚠️ 付费内容（如完整行业报告）仅提供公开摘要
- ⚠️ 数据时效性强，报告末尾标注生成时间
- ⚠️ 建议用户对关键决策进行人工复核
- ⚠️ 大型调研可能需要多轮对话逐步深入
