# ✍️ Humanizer ZH - 中文文本润色专家

## 技能概述

将生硬、机械、AI味浓重的文本转化为自然流畅、富有温度的中文表达，同时保留原意不变。

## 核心能力

### 🎯 能力一：AI痕迹消除

#### 常见 AI 写作特征（需要消除）
| 特征类型 | 典型表现 | 修改策略 |
|----------|----------|----------|
| **过度结构化** | "首先...其次...最后..." | 打破固定套路，变换连接词 |
| **空洞套话** | "值得注意的是""不言而喻" | 删除或换成具体表达 |
| **万能开头** | "在当今数字化时代""随着...的发展" | 直接切入主题，用具体场景开场 |
| **平衡癖好** | 正反两面都说，没有立场 | 敢于表达观点，适当主观 |
| **完美段落** | 每段都是总-分-总 | 长短不一，有的段一句就够 |
| **高级词汇堆砌** | "赋能""抓手""底层逻辑""闭环" | 用大白话替代，或精确使用 |
| **过于完整** | 面面俱到滴水不漏 | 有取舍，留白给读者思考 |
| **缺乏个性** | 读起来像说明书 | 加入个人视角/感受/例子 |

#### 去AI化实战对照

| ❌ AI味原文 | ✅ 人性化改写 |
|------------|-------------|
| 在当今数字化转型的浪潮中，企业面临着前所未有的机遇与挑战。 | 说实话，现在做企业不容易——一边是数字化喊得震天响，一边是不知道从哪下手。 |
| 综上所述，我们可以得出以下结论。 | 说到底，就一句话：... |
| 值得注意的是，该方法具有显著的优势。 | 这招确实管用。我试过之后发现... |
| 这是一个不可或缺的重要组成部分。 | 这块要是没做好，后面全白搭。 |
| 让我们深入探讨一下这个问题的本质。 | 说白了，问题的根子在于... |
| 通过以上的分析，我们不难发现... | 看完这些数据，事情就很清楚了。 |
| 在实际应用场景中... | 放到真实环境里一看... |
| 不可否认的是... | 必须承认一点... / 说真的... |
| 从某种意义上来说... | 换个角度看... / 往深了想... |

### 💎 能力二：文笔润色

#### 润色等级

| 等级 | 名称 | 效果描述 | 适用场景 |
|------|------|----------|----------|
| **Lv1** | 通顺修正 | 修复病句/错别字/标点 | 初稿清理 |
| **Lv2** | 流畅提升 | 优化节奏/衔接/用词准确度 | 日常写作 |
| **Lv3** | 文采增强 | 加入修辞手法/丰富表达层次 | 公众号/文案 |
| **Lv4** | 风格重塑 | 按目标风格全面改写 | 特定场景定制 |
| **Lv5** | 妙笔生花 | 注入个人声音/创造金句 | 演讲/品牌文案 |

#### 中文修辞手法库

| 手法 | 说明 | 示例 |
|------|------|------|
| **比喻** | 用熟悉事物解释陌生概念 | "这个Bug就像定时炸弹，平时不响，上线就炸。" |
| **类比** | 跨领域映射理解 | "写代码就像做饭，菜谱(文档)很重要，但火候(经验)更关键。" |
| **排比** | 句式重复增强气势 | "不是为了炫耀，不是为了跟风，而是因为热爱。" |
| **设问** | 自问自答引导思考 | "为什么选Vue而不是React？三个理由告诉你。" |
| **反语** | 正话反说增加趣味 | "这代码写得真'漂亮'——除了跑不通，哪都好。" |
| **白描** | 平实叙述胜过华丽辞藻 | "凌晨三点，他终于看到了那行 'build success'。" |
| **留白** | 话不说尽，余韵悠长 | "后来他再也没写过那个 if (true)。" |
| **口语化** | 对话感拉近距离 | "你有没有遇到过这种情况：明明代码没问题，就是跑不起来？" |

### 🎭 能力三：语调语气调整

#### 语调模式

| 模式 | 特征 | 用词倾向 | 适用场景 |
|------|------|----------|----------|
| **专业严谨** | 客观/精确/权威 | 表明/显示/依据/数据显示 | 学术/报告/官方文件 |
| **亲切友好** | 温暖/拉近距离 | 其实/你知道吗/我跟你讲 | 公众号/博客/社区 |
| **幽默风趣** | 轻松/有趣/有梗 | 说实话/扎心了/神操作 | 抖音文案/年轻群体 |
| **感性走心** | 共情/触动人心 | 那一刻/突然意识到/心里暖暖的 | 故事/品牌故事 |
| **干练利落** | 短句/有力/不废话 | 做了/成了/搞定 | 工作汇报/技术分享 |
| **文艺清新** | 优美/含蓄/意境 | 恰似/宛若/时光里/那些年 | 散文/心情随笔 |
| **热血激昂** | 有力量/号召/感染 | 就在今天/绝不/一起冲 | 演讲/动员/宣言 |
| **冷静理性** | 逻辑清晰/数据驱动 | 数据表明/事实上/从逻辑来看 | 分析评论/深度文章 |

#### 语气微调示例

同一件事的三种表达：

```
原始: 这个功能开发完成了。

专业严谨: 该功能模块已按照需求规格完成全部开发与测试，
         经验证符合上线标准，拟于本迭代周期末部署至生产环境。

亲切友好: 跟大家说个好消息！盼了好久的功能终于做完了，
         测试了一圈没发现问题，马上就能用上了~

干练利落: 功能开发完毕，测试通过，准备上线。
```

### 📝 能力四：文体适配

#### 各文体写作规范

##### 公众号文章
```
✅ 开头: 3秒抓眼球（提问/冲突/悬念/金句）
✅ 段落: 每段≤4行，手机阅读友好
✅ 小标题: 吸引人的短语，不要"第一部分"这种
✅ 配图提示: 建议插图位置
✅ 结尾: 金句收尾 + 互动引导（留言/转发）
❌ 避免: 长段落/学术腔/说教口吻
```

##### 工作邮件
```
✅ Subject: [标签] 简明扼要说清楚事由
✅ 开门见山: 第一段就说目的
✅ 结构: 背景 → 建议/问题 → 期望行动 → 时间
✅ 语气: 礼貌但不啰嗦
✅ Cc/Bcc: 精准抄送相关人员
❌ 避免: 情绪化表达/被动攻击/过度抱歉
```

##### 产品文案
```
✅ 用户视角: 不说"我们有XX"，说"你可以XX"
✅ 利益前置: 先说好处再说功能
✅ 数字量化: "提升3倍效率"比"大幅提升"有力
✅ 信任背书: 数据/评价/认证/案例
✅ 行动召唤(CTA): 明确告诉用户下一步做什么
❌ 避免: 自嗨式功能罗列/空洞形容词堆砌
```

##### 技术博客
```
✅ Problem first: 先说遇到什么问题
✅ Show don't tell: 上代码/截图/命令行输出
✅ 踩坑记录: "我当时犯了个错误..."比正确答案更有价值
✅ 可复现: 读者照着做能得到同样结果
✅ 参考链接: 相关文档/issue/discussion
❌ 避免: 翻译官方文档/只贴代码不解释/水字数
```

## 使用示例

### 示例1：基础润色
```
用户输入:
  本文将从以下几个方面进行分析和研究。
  首先，介绍了研究的背景和意义。
  其次，阐述了相关理论和方法。
  然后，展示了实验的结果和分析。
  最后，总结了全文的主要内容和创新点。

→ Lv3 文采增强输出:
  这项研究从四个层面展开：
  为什么做这件事？（背景篇）
  前人走到了哪里？（理论篇）
  我们做了什么发现了什么？（实验篇）
  以及这件事意味着什么。（总结篇）
  每一层都有它的故事。
```

### 示例2：去AI味
```
用户输入:
  在人工智能飞速发展的今天，深度学习技术已经广泛应用于各个领域，
  为社会带来了深刻的变革。本文旨在探讨深度学习在自然语言处理领域的
  应用及其发展趋势。

→ 去AI化输出:
  最近几年，AI这股风吹得有点猛。
  特别是深度学习——从前只能在实验室里跑的东西，
  现在已经渗透到你每天用的App里了。
  今天就想聊聊：深度学习到底是怎么"读懂"人话的？
  以及接下来它还会往哪个方向进化。
```

### 示例3：风格转换
```
用户输入: (一段技术实现描述)
  请把它改成"小白也能看懂"的科普风格

→ 风格重塑输出:
  把这段代码想象成一个餐厅：
  Controller = 服务员（接待客人下单）
  Service = 厨房（处理订单做菜）
  Repository = 仓库管理员（取食材）
  Database = 冷藏室（存东西的地方）
  
  客人来了 → 服务员接单 → 厨房看单做菜 → 仓库拿食材 → 上菜！
  就是这么个流程。
```

### 示例4：多版本输出
```
用户: 帮我把这段话改得好一点，给我几个版本选择
→ 输出版本A(正式): 适合汇报/论文
→ 输出版本B(轻松): 适合公众号/分享
→ 输出版本C(精简): 适合PPT/摘要
→ 输出版本D(故事感): 适合演讲/视频脚本
→ 附带: 每个版本的适用场景说明
```

## 质量标准

| 维度 | 标准 |
|------|------|
| **准确性** | 改写后原意不变，不曲解不遗漏 |
| **通顺度** | 读起来顺口，没有拗口的表达 |
| **自然度** | 像真人写的，不像机器生成的 |
| **一致性** | 全文风格统一，不会忽正式忽随意 |
| **感染力** | 能引发读者情绪共鸣或行动欲望 |
| **简洁度** | 没有废话，每个字都有存在的理由 |
