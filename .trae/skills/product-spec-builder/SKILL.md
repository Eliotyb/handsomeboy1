# 📋 Product Spec Builder - 产品需求文档生成器

## 技能概述

通过结构化引导，自动生成专业级产品需求文档（PRD），覆盖从需求收集到验收标准的完整文档链。

## 核心能力

### 1. 文档类型支持
| 文档类型 | 说明 |
|----------|------|
| **完整 PRD** | 包含所有章节的标准产品需求文档 |
| **BRD (商业需求)** | 商业价值/市场分析/ROI评估 |
| **MRD (市场需求)** | 市场调研/用户画像/竞品分析 |
| **功能规格** | 单功能点的详细技术规格 |
| **用户故事** | Agile格式的User Story + 验收标准 |
| **API 接口文档** | RESTful API 规格定义 |
| **数据模型设计** | ER图/表结构/字段定义 |

### 2. PRD 完整结构模板

```markdown
# [产品名称] 产品需求文档 (PRD)

## 1. 文档信息
- 版本号 / 作者 / 日期 / 状态(草案/评审/定稿)

## 2. 项目背景与目标
- 业务背景 / 痛点分析 / 项目目标(SMART原则)

## 3. 用户角色与画像
- 目标用户群体 / 用户画像卡片 / 用户使用场景

## 4. 功能范围
- MVP范围定义 / V1.0功能清单 / 后续版本规划
- 功能优先级排序(P0/P1/P2)

## 5. 功能详细需求
### 5.1 [模块名]
- 功能描述
- 前置条件
- 主流程(正常路径)
- 异常流程(边界情况)
- UI原型描述 / 交互说明
- 数据要求(输入/输出/存储)
- 非功能性需求(性能/安全/兼容性)
- 验收标准(Acceptance Criteria)

## 6. 信息架构
- 页面结构 / 导航逻辑 / 信息层级

## 7. 数据需求
- 数据实体 / 字段定义 / 状态机 / 关系

## 8. 非功能需求
- 性能指标 / 安全要求 / 兼容性 / 可用性

## 9. 上线计划与风险
- 里程碑 / 风险清单 / 应对方案

## 附录
- 术语表 / 参考文档 / 变更记录
```

## 工作流程

```
Step 1: 需求采集（交互式提问）
  ↓ "你要做什么产品？解决什么问题？"
Step 2: 结构化整理
  ↓ 自动归类到PRD各章节
Step 3: 补全追问
  ↓ 发现缺失信息时主动询问
Step 4: 文档生成
  ↓ 输出完整Markdown格式PRD
Step 5: 质量检查
  ↓ 检查完整性/一致性/可测试性
Step 6: 迭代优化
  ↓ 根据反馈修改特定章节
```

## 使用示例

### 示例1：从零开始创建PRD
```
用户: 我想做一个校园二手交易平台
→ 引导提问:
  Q1: 目标用户是谁？(全校学生? 仅限本校?)
  Q2: 核心交易品类?(书籍/电子产品/生活用品?)
  Q3: 差异化优势?(相比闲鱼/转转?)
  Q4: 盈利模式?(免费/佣金/广告?)
  ... (5-8个关键问题)
→ 输出: 完整PRD文档(含功能列表+用户故事+数据模型)
```

### 示例2：补充现有文档
```
用户: 这是我的功能列表，帮我补全PRD其他部分
→ 分析已有内容 → 识别缺失章节 → 补充:
  - 用户角色定义
  - 非功能需求
  - 验收标准
  - 数据模型
  - 上线计划
```

### 示例3：输出用户故事(Agile)
```
用户: 给我Agile格式的用户故事
→ 输出格式:
  ## US-001: 用户发布商品
  **作为** 二手买家/卖家
  **我希望** 能够发布待售商品，包含图片和描述
  **以便** 其他用户可以看到并购买

  **验收标准:**
  - [ ] 支持上传最多9张图片
  - [ ] 必填项：标题/价格/分类/描述
  - [ ] 价格范围：0.01-99999元
  - [ ] 发布后状态为"上架中"
  ...
```

## 质量标准

| 检查项 | 标准 |
|--------|------|
| **完整性** | 每个功能点都有前置/主流程/异常/验收标准 |
| **可测试性** | 验收标准必须是可量化/可验证的 |
| **无歧义** | 避免"可能""大概""较好"等模糊词汇 |
| **一致性** | 术语/命名/流程前后统一 |
| **可行性** | 技术上可实现，资源上可落地 |

## 输出格式选项

- `markdown` - 默认，适合文档阅读
- `confluence` - Confluence Wiki 格式
- `notion` - Notion 数据库格式
- `jira` - Jira Epic/Story 格式
- `table` - Excel 可导入的表格格式
