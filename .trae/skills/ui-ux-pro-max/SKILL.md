# 🏆 UI/UX Pro Max - 全栈设计专家

## 技能概述

提供从设计战略到像素级细节的全链路UI/UX设计服务，包含设计系统搭建、交互方案制定、可用性审计等专业能力。

## 核心服务矩阵

### 🎯 服务一：设计系统构建 (Design System)

#### 1.1 色彩系统
```
色彩体系:
├── Primary (主色)
│   ├── Primary-50  ~  Primary-900 (10级色阶)
│   └── Primary Light / Dark 变体
├── Semantic (语义色)
│   ├── Success (成功/正向) - 绿色系
│   ├── Warning (警告/注意) - 黄橙色系
│   ├── Error (错误/危险) - 红色系
│   └── Info (信息/中性) - 蓝色系
├── Neutral (中性色)
│   ├── Gray-50  ~  Gray-900 (灰度阶梯)
│   └── Text primary/secondary/tertiary/disabled
└── Dark Mode 映射规则
    └── 每个色值的暗色对应值
```

#### 1.2 字体排版系统
```
Typography Scale ( Modular Scale ):
├── Display (展示): 48px/ Bold / Heading
├── H1 (一级标题): 36px / Semibold
├── H2 (二级标题): 28px / Semibold  
├── H3 (三级标题): 22px / Medium
├── Body Large (正文大): 17px / Regular
├── Body (正文): 15px / Regular
├── Caption (说明): 13px / Regular
└── Label (标签): 12px / Medium

行高比例: 1.2(标题) / 1.5(正文) / 1.0(标签)
字间距: -0.02em(大标题) / 0(正文) / 0.02em(大写英文)
```

#### 1.3 间距系统 (Spacing)
```
Base Unit: 4px
├── xs: 4px   (元素内微间距)
├── sm: 8px   (紧密关联元素)
├── md: 16px  (常规间距)
├── lg: 24px  (区块间距)
├── xl: 32px  (大区块分隔)
└── xxl: 48px+ (页面级分隔)

Grid: 12列栅格, Gutter: 16-24px, Margin: 16-24px
```

#### 1.4 组件规范
- **按钮**: Primary/Secondary/Tertiary/Ghost/Danger × 3尺寸
- **输入框**: Default/Error/Disabled/带图标 × 多种变体
- **卡片**: 基础卡/统计卡/列表卡/媒体卡
- **导航**: 顶导/侧边栏/底Tab/面包屑
- **反馈**: Toast/Alert/Modal/Drawer/Popover
- **数据展示**: Table/List/Empty/Loading/Skeleton

### 🔀 服务二：交互方案设计 (Interaction Design)

#### 2.1 用户旅程地图
```
用户阶段: 认知 → 探索 → 尝试 → 使用 → 留存 → 推荐
每个阶段:
  - 用户目标
  - 触发场景
  - 行为路径
  - 痛点/机会点
  - 情绪曲线
  - 接触点(Touchpoint)
```

#### 2.2 核心交互模式
| 模式 | 适用场景 | 设计要点 |
|------|----------|----------|
| **向导式** | 复杂表单/新手引导 | 进度指示/步骤拆分/可回退 |
| **仪表盘** | 数据监控/管理后台 | 信息层级/KPI突出/钻取 |
| **列表-详情** | 内容浏览/数据查询 | 筛选/排序/分页/快速预览 |
| **F型布局** | 信息阅读/新闻feed | 左侧导航/顶部重要/右侧辅助 |
| **卡片瀑布** | 商品浏览/灵感发现 | 无限滚动/懒加载/兴趣推荐 |
| **对话式** | 智能助手/客服 | 气泡/输入区/快捷回复 |

#### 2.3 动效规范
```
时长标准:
├── Micro (微交互): 100-200ms (hover/press/toggle)
├── Standard (标准): 200-300ms (展开/收起/切换)
├── Expressive (表达): 300-500ms (模态框/页面转场)
└── Hero (重磅): 500-800ms (首次入场/特殊强调)

缓动函数:
├── ease-out: 进入视口 (快速开始→慢结束)
├── ease-in: 离开视口 (慢开始→快速消失)
├── ease-in-out: 状态变化 (平滑过渡)
└── spring: 物理交互 (弹性/惯性)
```

### 🔍 服务三：可用性审查 (UX Audit)

#### 3.1 十大可用性 heuristic 评估

| # | 原则 | 检查要点 |
|---|------|----------|
| 1 | **系统状态可见** | 加载状态/进度反馈/操作结果是否明确 |
| 2 | **环境贴切** | 用词是否符合用户心智模型 |
| 3 | **用户可控** | 能否撤销/退出/取消操作 |
| 4 | **一致性标准** | 相同功能在不同位置表现一致吗 |
| 5 | **防错设计** | 是否在错误发生前预防 |
| 6 | **识别优于记忆** | 操作选项是否可见/可识别 |
| 7 | **灵活高效** | 新手友好同时专家有快捷方式吗 |
| 8 | **美学简约** | 无无关信息，视觉层次清晰 |
| 9 | **错误恢复** | 错误信息是否明确/提供解决方案 |
| 10 | **帮助文档** | 必要时能否轻松获取帮助 |

#### 3.2 输出报告模板
```
## UX 审查报告 - [产品名]

### 总体评分: X/10

### 🔴 严重问题 (P0 - 必须修复)
1. [问题描述] → 影响[X%用户] → 建议[解决方案]

### 🟡 中等问题 (P1 - 应该修复)
...

### 🟢 优化建议 (P2 - 可以改进)
...

### 📊 各维度评分雷达图
- 可学习性: X/10
- 效率性: X/10
- 可记忆性: X/10
- 错误率: X/10
- 满意度: X/10
```

## 使用示例

### 示例1：从零搭建设计系统
```
用户: 帮我们的SaaS产品搭建一套设计系统
→ 输出:
  1. Brand Color Palette (主色+扩展色板+暗色映射)
  2. Typography System (字体/字号/字重/行高)
  3. Spacing & Grid (间距/栅格/断点)
  4. Component Library (20+基础组件规范)
  5. Icon System (图标风格/尺寸/使用规则)
  6. Token Output (CSS Variables / Design Tokens JSON)
```

### 示例2：交互方案评审
```
用户: 这个注册流程体验不好，帮我优化
→ 分析:
  当前: 3页表单 → 每页5-8字段 → 无进度提示
  问题: 认知负担重/放弃率高/无法预览后续
→ 方案:
  1. 单页长表单(分区折叠) + 实时验证
  2. 或: 保持分步但增加步骤条+每步≤3字段
  3. 增加: 社交账号一键注册入口
  4. 微交互: 字段聚焦动画/完成勾选动效
```

### 示例3：A/B测试方案
```
用户: 这个转化按钮我想做A/B测试
→ 输出测试方案:
  变量: 按钮文案/颜色/尺寸/位置
  指标: CTR(点击率)/CVR(转化率)/停留时间
  样本量: 每组≥1000曝光(95%置信度)
  时长: 运行至少7天(覆盖完整周期)
  统计方法: 卡方检验 / t检验
```
