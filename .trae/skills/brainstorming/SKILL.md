# 💡 Brainstorming - 九步头脑风暴引擎

## 技能概述

系统化的九步创意方法论，帮助用户从模糊的想法出发，经过结构化思维过程，产出可执行的创意方案。

## 🌀 九步头脑风暴法

```
Step 1: 明确定义 ────────────── ✅ 问题是什么？
   ↓
Step 2: 拆解要素 ────────────── 🔍 可以拆成哪些维度？
   ↓
Step 3: 自由发散 ────────────── 💨 不评判，先疯狂输出
   ↓
Step 4: 分类聚类 ────────────── 📂 把相似想法归到一起
   ↓
Step 5: 建立连接 ────────────── 🔗 跨类别组合出新点子
   ↓  
Step 6: 深度追问 ────────────── ❓ "如果...会怎样？""为什么不是..."
   ↓
Step 7: 筛选评估 ────────────── ⚖️ 可行性/价值/成本/时间 四维打分
   ↓
Step 8: 方案具象 ────────────── 🎯 从抽象概念变成具体行动
   ↓
Step 9: 行动计划 ────────────── 📋 第一步做什么？谁来做？何时完成？
```

---

### Step 1: 明确定义 (Define)

**目标**: 用一句话精准描述要解决的问题/机会

**引导问题**:
- 你想解决什么具体问题？
- 理想的最终结果是什么样的？
- 有什么约束条件？（预算/时间/技术/人力）
- 成功的标准是什么？

**输出模板**:
```
📌 核心命题: [一句话]
🎯 目标: [可衡量的目标]
⛓️ 约束: [限制条件列表]
✅ 成功标准: [如何判断成功]
```

**示例**:
```
📌 核心命题: 如何让大学生更愿意参与社团活动？
🎯 目标: 社团活动参与率提升50%
⛓️ 约束: 预算<5000元/2个月内见效/不能占用上课时间
✅ 成功标准: 活动报名人数/社团活跃人数双增长
```

---

### Step 2: 拆解要素 (Decompose)

**目标**: 将大问题拆解为可操作的子维度

**常用拆解框架**:

| 场景 | 推荐拆解维度 |
|------|-------------|
| **产品创新** | 用户/场景/痛点/功能/渠道/盈利 |
| **活动策划** | 主题/形式/时间/地点/人群/宣传/预算 |
| **内容创作** | 受众/目的/形式/风格/渠道/频次 |
| **解决问题** | 原因/影响/相关方/资源/风险/时机 |
| **学习规划** | 目标/现状/差距/方法/资源/反馈 |

**工具**: 思维导图 / 鱼骨图 / 5W1H

---

### Step 3: 自由发散 (Diverge)

**核心规则**: 
- ⭐ **不评判!** 再离谱的想法也先记录
- ⭐ **追求数量** 先数量后质量（目标30+个想法）
- ⭐ **站在他人角度** 换位思考
- ⭐ **组合旧元素** 新想法=旧元素的新组合

**发散技巧库**:

| 技巧 | 说明 | 触发词 |
|------|------|--------|
| **SCAMPER** | 替代/合并/借鉴/修改/另用/消除/重排 | "能不能把X换成Y？" |
| **逆向思维** | 反过来想会怎样？ | "如果不让XX，而是强制YY呢？" |
| **类比迁移** | 其他领域怎么解决的？ | "这像不像餐厅排队的问题？" |
| **极端假设** | 放大到无限或缩小到零 | "如果预算无限/只有1块钱呢？" |
| **角色扮演** | 换成不同身份思考 | "如果你是大学生/校长/赞助商会怎么想？" |
| **随机刺激** | 随机词汇联想 | 随机一个词：🎲"咖啡" → 和我们的问题有什么关系？ |

---

### Step 4: 分类聚类 (Cluster)

**方法**: 将 Step 3 的所有想法按自然归属分组

**分类维度参考**:
- 按类型分：功能类 / 体验类 / 运营类 / 技术类
- 按难度分：简单快速(Quick Win) / 中等投入 / 长期战略
- 按受众分：面向用户 / 面向运营 / 面向管理
- 按阶段分：短期(1周内) / 中期(1月内) / 长期(季度)

**输出**: 分类后的想法矩阵 + 每类的核心主题

---

### Step 5: 建立连接 (Connect)

**核心操作**: 跨类别组合，产生新创意

**组合公式**:
```
[A类别的想法] + [B类别的想法] = [全新创意]
```

**示例**:
```
[线上直播] + [游戏化积分] = 直播答题赢社团积分
[校友分享] + [技能交换] = 校友导师1v1技能工坊
[盲盒玩法] + [社团招新] = 社团体验盲盒盒
```

**强制连接练习**: 随机抽取两个不同类别的想法，强行找结合点

---

### Step 6: 深度追问 (Challenge)

**对每个有潜力的想法进行压力测试**:

| 追问方向 | 典型问题 |
|----------|----------|
| **可行性** | 技术上能实现吗？需要什么资源？ |
| **必要性** | 真的解决了用户痛点吗？还是自嗨？ |
| **差异化** | 别人做过吗？我们能做得不同吗？ |
| **可持续性** | 能长期运行吗？还是一次性活动？ |
| **放大/缩小** | 如果规模扩大10倍会怎样？缩小到最小可行版本呢？ |
| **最坏情况** | 最可能失败的原因是什么？如何预防？ |

**工具**: 5 Why 法（连续问5次"为什么"找到根因）

---

### Step 7: 筛选评估 (Evaluate)

**四维评分模型**:

| 维度 | 权重 | 评分标准(1-5分) |
|------|------|-----------------|
| **价值 Impact** | 30% | 对目标的贡献程度 |
| **可行 Feasibility** | 25% | 技术/资源可实现性 |
| **成本 Cost** | 20% | 时间/金钱/人力投入 |
| **独特 Novelty** | 15% | 差异化/创新程度 |

**评分模板**:
```
┌────────────────┬──────┬──────┬──────┬──────┬──────┐
│ 想法           │ 价值 │ 可行 │ 成本 │ 独特 │ 总分 │
├────────────────┼──────┼──────┼──────┼──────┼──────┤
│ 想法A          │  5   │  4   │  3   │  4   │ 4.05 │
│ 想法B          │  4   │  5   │  5   │  2   │ 4.00 │
│ 想法C          │  5   │  3   │  2   │  5   │ 3.70 │
└────────────────┴──────┴──────┴──────┴──────┴──────┘
→ 推荐: 想法A > 想法B > 想法C
```

---

### Step 8: 方案具象 (Concrete)

**将抽象想法转化为具体方案**:

```
【想法名称】一句话描述

📋 方案概述 (Elevator Pitch, 30秒能说清)
👥 目标用户 (Who)
💡 核心价值 (What - 3个关键卖点)
🔄 运作方式 (How - 关键流程)
🆚 与现有方案的差异 (Why Us)
📊 预期效果 (Expected Outcome)
```

---

### Step 9: 行动计划 (Action Plan)

**OKR 式行动计划**:

```markdown
## Objective: [总体目标]

### Key Result 1: [KR1]
- Task 1.1: [具体动作] → 负责: XX → 截止: MM-DD
- Task 1.2: [具体动作] → 负责: XX → 截止: MM-DD

### Key Result 2: [KR2]
- Task 2.1: [具体动作] → 负责: XX → 截止: MM-DD

### Milestones:
- 🏁 Week 1: [里程碑1]
- 🏁 Week 2: [里程碑2]
- 🏁 Week 4: [最终交付]

### Risk & Mitigation:
- ⚠️ 风险1 → 应对策略
```

## 使用示例

### 完整案例：策划一场校园社团招新活动

```
用户: 我们社团要招新了，帮我 brainstorm一下怎么搞

→ Step1 定义: 目标=招新100人, 约束=2周, 预算3000元
→ Step2 拆解: 宣传渠道/活动形式/ incentives/转化路径
→ Step3 发散: 35个想法(线下摆摊/抖音挑战/学长代言/...)
→ Step4 分类: 线上宣传(8)/线下活动(12)/激励政策(7)/...
→ Step5 连接: "抖音挑战" + "学分奖励" = 完任务拿学分
→ Step6 追问: 学分谁批? → 找团委合作置换志愿时长
→ Step7 评分: Top3 = 线下体验日/社群裂变/联名活动
→ Step8 具象: 详细执行方案
→ Step9 计划: 每日任务清单 + 责任人 + Deadline
```

## 注意事项

- ⚠️ 头脑风暴的核心是**数量**而非质量，先多后精
- ⚠️ 不要在 Step 3 发散阶段打断或评判任何想法
- ⚠️ 记录所有想法，即使当时觉得没用，后续步骤可能激活
- ⚠️ 团队头脑风暴时建议用共享文档实时记录
- ⚠️ 如果卡住了，使用**随机刺激**技巧打破思维定势
