# 🎬 PPT Generator - 专业PPT自动生成器

## 技能概述

根据用户提供的主题和素材，自动生成结构完整、排版专业的 PowerPoint 演示文稿（以可直接使用的 HTML/PPTX 格式输出）。

## PPT 结构设计体系

### 📐 标准PPT结构模板

```
Slide 1:   封面页 (Title Slide)
            ├─ 主标题 + 副标题 + 演讲者 + 日期 + Logo
            └─ 设计: 大气简洁/视觉冲击/品牌色

Slide 2:   目录页 (Agenda/Contents)
            ├─ 4-6个主要章节
            └─ 设计: 清晰导航/编号/图标点缀

Slide 3-N: 内容页 (Content Slides)
            ├─ 每章 3-8 页
            ├─ 类型交替: 文字/图表/图片/对比/时间线
            └─ 设计: 统一风格/渐变过渡

Slide N+1: 总结页 (Summary/Takeaway)
            ├─ 核心要点回顾 (3-5条)
            └─ 设计: 重点突出/记忆锚点

Slide N+2: 结束页 (Q&A / Thank You)
            ├─ 致谢 + 联系方式 + QR码
            └─ 设计: 与封面呼应/留白优雅
```

### 🎨 幻灯片类型库

#### 文字类
| 类型 | 布局 | 适用 |
|------|------|------|
| **纯文字页** | 标题+3-5条要点 | 概念介绍/原则列举 |
| **左右分栏** | 左文右图/左图右文 | 对比说明/图文并茂 |
| **三栏布局** | 并列三个板块 | 平行概念/三维分析 |
| **引用页** | 大号引言+出处 | 金句强调/观点引用 |
| **定义页** | 术语+解释+示例 | 概念解释/知识科普 |

#### 数据类
| 类型 | 布局 | 适用 |
|------|------|------|
| **数据大屏** | KPI数字+趋势箭头 | 关键指标展示 |
| **柱状图页** | 图表+洞察文字 | 排名/对比/趋势 |
| **饼图页** | 饼图+细分说明 | 占比/构成分析 |
| **表格页** | 精简表格+高亮 | 数据明细/参数对比 |
| **流程图页** | 步骤流程+说明 | 过程展示/方法论 |

#### 视觉类
| 类型 | 布局 | 适用 |
|------|------|------|
| **全屏大图** | 高质量背景图+少量文字 | 氛围营造/情感共鸣 |
| **图片网格** | 2x2/3x3图片矩阵 | 作品展示/案例集合 |
| **时间线** | 横向/纵向时间轴 | 发展历程/项目节点 |
| **架构图** | 层级/网状/循环架构 | 系统结构/组织架构 |
| **地图页** | 地图+标注点 | 区域分析/覆盖范围 |

## PPT 场景模板

### 📊 场景1: 工作汇报 / 周报月报
```
建议页数: 8-15页
结构:
  [封面] → [目录] → [工作概览] → [重点项目进展(3-5页)] 
  → [数据指标] → [问题与风险] → [下月计划] → [总结]

每页要点不超过 6 条
数据用图表说话，不用纯数字堆砌
问题页同步给出解决方案
```

### 💰 场景2: 商业计划书 BP
```
建议页数: 15-25页
结构:
  [封面] → [痛点] → [解决方案] → [市场规模] → [产品展示]
  → [商业模式] → [竞争分析] → [运营数据] → [团队介绍]
  → [财务预测] → [融资需求] → [愿景] → [结束]

重点: 市场规模用权威数据背书
      竞争用矩阵图差异化呈现
      财务用三年预测表
```

### 🎓 场景3: 学术答辩 / 毕业答辩
```
建议页数: 20-30页
结构:
  [封面] → [目录] → [研究背景] → [文献综述] → [研究方法]
  → [核心技术/算法] → [实验设计] → [结果分析(多页)]
  → [讨论] → [创新点] → [总结展望] → [致谢] → [问答]

重点: 方法论页放流程图
      结果页用图表+数据表
      每页有明确的小结句
```

### 🏆 场景4: 年终总结 / 述职报告
```
建议页数: 12-20页
结构:
  [封面] → [年度关键词] → [核心业绩(KPI达成)]
  → [重点项目回顾(3-5个)] → [数据亮点] 
  → [成长与收获] → [不足与反思] → [明年规划] → [致谢]

技巧: 用时间线串起全年大事
      用前后对比体现成长
      用SMART原则写明年目标
```

### 📚 场景5: 培训课件 / 课程PPT
```
建议页数: 30-60页（按1小时/15-20页估算）
结构:
  [封面] → [课程目标] → [目录] → [章节内容(每个概念1-2页)]
  → [案例分析] → [互动练习] → [知识总结] → [课后作业] → [Q&A]

技巧: 每章开始有过渡页
      重要概念单独一页+配图
      每10页安排一个互动环节
      结尾提供知识检查清单
```

## 使用示例

### 示例1：快速生成
```
用户: 帮我做一个"2025年Q1销售总结"PPT
→ 交互确认:
  Q1: 听众是谁?(老板/团队/客户?)
  Q2: 多少页合适?(10页左右?)
  Q3: 风格偏好?(商务/活泼?)
  Q4: 有数据源吗?(Excel/手动输入?)
→ 输出: 完整HTML格式PPT(可直接浏览器打开全屏播放)
```

### 示例2：基于大纲生成
```
用户: 这是我的大纲，帮我做成PPT
大纲:
  1. 项目背景
  2. 技术架构(Spring Boot + Vue3)
  3. 核心功能模块
  4. 数据库设计
  5. 部署方案
  6. Q&A
→ 为每章分配合适的幻灯片类型:
  Ch1: 引用页(项目slogan) + 定义页
  Ch2: 架构图页 + 技术栈表格
  Ch3: 功能截图网格(3x2) + 流程图
  Ch4: ER图 + 字段说明表
  Ch5: Docker架构图 + 部署步骤时间线
  Ch6: 致谢页
```

### 示例3：迭代优化
```
用户: 这个PPT太素了，加点设计感
→ 优化操作:
  - 封面: 加渐变背景 + 几何装饰元素
  - 数据页: 柱状图改渐变色 + 加动画效果建议
  - 文字页: 要点加图标前缀 + 左侧色条装饰
  - 过渡: 章节间加过渡页(大号章节名)
  - 配色: 统一为主题色板(主色+辅色+中性色)
```

## 质量检查清单

- [ ] 每页只有一个核心观点（一页一事）
- [ ] 字号层次分明（标题≥32pt / 正文≥18pt / 注释≥12pt）
- [ ] 行距舒适（1.2-1.5倍）
- [ ] 对齐严格（左对齐/居中不混用）
- [ ] 配色不超过3种主色
- [ ] 图片高清无水印
- [ ] 数据来源标注
- [ ] 无错别字/语法错误
- [ ] 动画适度（不过度花哨影响阅读）
- [ ] 总时长匹配演讲时间（约1-2分钟/页）

## 输出格式

| 格式 | 说明 | 使用方式 |
|------|------|----------|
| **HTML** | 默认输出 | 浏览器直接打开/F11全屏播放 |
| **PPTX大纲** | 结构文本 | 导入PowerPoint后再美化 |
| **Markdown** | 逐页内容 | 配合Marp/Slidev转为PPT |
| **详细脚本** | 每页备注 | 含演讲者备注/计时建议 |
